It's not quite as simple as we thought. here are all the dashes with
the surrounding words from a randomly chosen file (well, the first
file, so not actually randomly chosen, but I think probably quite
reopresentative). We were going to say that if MADAMIRA found no
solution for the word before the dash then if it was a prefix of the
following word then we would use the diacritics from the following one
on the grounds that it was almost certainly a false start for the
following word and hence would be phonetically the same. But MADAMIRA
actually finds a solution in a very large percentage of cases. It's
just usually the wrong solution. The ones below that are marked with
**** are the ones where the word before the dash is a prefix of the
following one.

(2) is an example where MADAMIRA didn't recognise the preceding word
but it was a prefix of the following one. Then (3)-(6), (8), (12) are
places where it did think it had recognised the previous one but it
clearly got it wrong. And then there are lots where the word before
the dash was "Al", e.g. (7), (10), (11): for some of these the
following word does start with "Al", for some it doesn't, but I think
that in every case the item before the dash was intended to be "Al".

There are only three out of 46 where MADAMIRA didn't think it had
recognised the preceding word, and for two of these the preceding word
is a prefix for the following one.

So I think we say: if the preceding word is a prefix of the following
one, whether or not MADAMIRA thinks it recognised it, we borrow the
diacritics from the following one; and in every case we insert a short
pause between the two.
